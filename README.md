# codegen-standards

Production-grade стандарты для AI-агентов, генерирующих код.

## О репозитории

`codegen-standards` — это набор skills и правил, которые задают единый инженерный стандарт для AI-генерации.

Репозиторий нужен, чтобы AI:
- писал код предсказуемо и последовательно;
- соблюдал архитектурные ограничения проекта;
- выдавал результат, пригодный для поддержки и production.

## Зачем это нужно

Без стандартов AI часто генерирует разнородные решения: разный стиль, случайные архитектурные ходы, неполные реализации.

С этим репозиторием команда получает:
- единое качество кода между людьми и агентами;
- меньше правок после генерации;
- прозрачные правила, которые можно обсуждать, версионировать и улучшать.

## Как использовать

Подключайте skills явно перед генерацией кода и указывайте, что правила обязательны к соблюдению.

Рекомендуемый процесс:
1. Определите контекст задачи (стек, слой, тип фичи).
2. Подключите релевантные skills для этого контекста.
3. Зафиксируйте приоритет: архитектура, потом реализация, потом стиль/линтинг.
4. Проверьте результат на соответствие правилам перед merge.

Важно: skills не заменяют ревью. Они уменьшают вариативность и повышают качество первого результата.

## Принципы

- Architecture-first: сначала границы и ответственность модулей, потом код.
- Explicit rules: требования формулируются как проверяемые правила.
- Deterministic output: одинаковый контекст должен давать близкий результат.
- Senior mindset: решения с учетом поддержки, масштабирования и DX.

## Кому подходит

- Командам, где AI участвует в ежедневной разработке.
- Проектам с высокими требованиями к качеству и консистентности.
- Техлидам, которым нужен управляемый стиль AI-генерации.

## Вклад

Если добавляете новые skills:
- пишите правила как конкретные требования;
- убирайте двусмысленность и дублирование;
- проверяйте совместимость с уже существующими стандартами;
- обновляйте документацию по процессу использования.
